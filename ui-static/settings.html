Wi-Fi connection settings

By default OPCB works in the Access Point mode. In this mode it creates its own Wi-Fi network for an easy access to this web-interface. If you can't find your device after switching it to the Wi-Fi client mode, insert the USB flash stick with an empty file named RESET_CONF to restore default settings.

Please consult the documentation for more details: https://docs.overvis.com/#/OPCB/User_Docs/Guides/Web_Interface

Internal Linux interface to use for Wi-Fi connection. Default value (wlan0) should be changed only if you are using external (USB) Wi-Fi modems.

Wi-Fi client - connect to an existing Wi-Fi station.

Wi-Fi access point - device will create its own Wi-Fi network for easy access to this web-interface locally (enabled by default).

After connecting to the OPCB access point, open http://setup.overvis.com/ to see this page. In access point mode, the internet connection may only be provided through Ethernet.

If you don't need Wi-Fi access, it's better to disable the Wi-Fi completely because access point mode does not require authentication.

Modbus TCP Server settings

As a Modbus TCP server, OPCB translates incoming Modbus requests to the connected Modbus RTU or Modbus TCP clients (slaves). OPCB can operate as a Modbus TCP server and client simultaneously. To disable server mode completely, select "Disabled" in the network interface option.

OPCB Modbus TCP server is accessible by device's IP address and specified port.
Timeout in milliseconds to wait for the response for the request sent to any of the Modbus slaves. If the slave doesn't respond in the specified time limit, the OPCB Modbus server will return "Gateway Target Device Failed to Respond" Modbus error (code 11) to the requester side.
Maximum number of requests simultaneously accepted from the clients. 1 - each request awaits until the previous request finishes. Other values allow the requests to be sent in groups. Higher values may improve communication speed, but may also cause problems with some of the connected devices that may have processing memory limitations.

Modbus TCP Client settings

If the network interface in the client form below is not "Disabled", OPCB will try to connect to the given IP address/port and use this TCP connection as a gateway to Modbus slave devices in the specified Modbus address range. That means that OPCB will translate all incoming requests with the device addresses (UIDs) fitting this range into this connection, translating them to the Modbus TCP format if needed.

Client #1

Requests with those Modbus UIDs (device addresses) will be translated into this client's connection. Specify as ranges of numbers from 0 to 255 separated by commas. Example: 0-25,50-255. If the ranges intersect with other clients (TCP or RTU), requests will be sent to all of them.
If the connection is interrupted or can't be established, OPCB will wait for specified period before trying to re-establish it.
Timeout in milliseconds to wait for the response for the request sent to the server. If the server doesn't respond in the specified time limit, the OPCB will return "Gateway Target Device Failed to Respond" Modbus error (code 11) to the requester side.

RS-485 Modbus settings

OPCB connects to the Modbus RTU/ASCII devices via RS-485 interface. Interface is created automatically after insertion of the USB<>RS-485 modem. If you don't see the interface in the dropdown, insert the modem and click "Refresh" button.

Interface may change its name if the modem is unplugged and re-inserted into the USB socket. In such case please refresh the dropdown list and select a new name.

Port #1

Name of the Linux device that corresponds to this RS-485 port. Device is created automatically after insertion of USB<>RS-485 modem.
In master mode OPCB translates incoming requests to this RS-485 network. In slave mode OPCB awaits for the requests from this network and translates them to any TCP/RTU slave interfaces.
Requests with those Modbus UIDs (device addresses) will be translated into this RS-485 network. Specify as ranges of numbers from 0 to 255 separated by commas. Example: 0-25,50-255. If the ranges intersect with other clients (TCP or RTU), requests will be sent to all of them.

In master mode - time to wait for the response for the request sent to this RS-485 network. If there is no response in the specified time limit, the OPCB will return "Gateway Target Device Failed to Respond" Modbus error (code 11) to the requester side.

In slave mode - time to wait for the slave devices in the other networks to respond after the request translation.
